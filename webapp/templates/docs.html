Product Guide

How Griders Works

What Griders Does

Griders connects TradingView signals with Cryptorg futures Ghost Bot trading. It receives strategy alerts, checks the selected connection, applies user plan limits and safety filters, then sends open/update/close commands to Cryptorg through the Ghost Bot webhook.

Griders uses the read-only Cryptorg Bybit Liquidity API key to read balance, open positions, orders, and closed PnL. Trading commands are sent only through the Ghost Bot webhook configured in the connection.

Plans

All public plans use GRID DCA 2.9. Higher plans expand simultaneous deal limits, first-order limits, available pairs, and volume calculation modes.

Free: active for 14 days, recommended deposit from 50 USDT, limits 4 / 4 / 4, first order up to 6 USDT, manual first-order input, BTCUSDT, ETHUSDT, and SOLUSDT unavailable.

Free Plus: no time limit, recommended deposit from 100 USDT, limits 6 / 6 / 6, first order up to 12 USDT, first order can be calculated from 1% to 10% of deposit. Requires registration through the Cryptorg referral link and connecting the new referred account to Griders.

Start: recommended deposit from 500 USDT, limits 8 / 8 / 8, first order up to 60 USDT, BTCUSDT unavailable. Start is paid through Tribute: 10 USDT/month.

Start Plus: recommended deposit from 1000 USDT, limits 10 / 10 / 10, first order up to 120 USDT, all pairs including BTCUSDT. Requires active Start and a new referred Cryptorg account.

Premium: limits 12 / 12 / 12, first order up to 600 USDT, all GRID DCA pairs. Premium is paid through Tribute: 50 USDT/month.

Premium Plus: limits 40 / 20 / 20, first order up to 2000 USDT, first order from deposit up to 15%, all pairs. Requires Premium and a new referred Cryptorg account.

Paid Plan Activation

  1. Open Profile and enter your Telegram username without the @ sign.
  2. Confirm the Telegram account through @griders_tarif_bot.
  3. Open Plans and connect the selected paid plan.
  4. For Start or Premium, complete the Tribute subscription.
  5. For Plus plans, register through the Cryptorg referral link and connect the new referred account to Griders trading.

Telegram binding is required to enable paid plans. If a paid plan cannot be verified, access can return to the Free plan until verification is restored.

Account Security

In Profile you can set a nickname, link Telegram, change email, choose a time zone, request a password reset, and enable two-factor authentication.

Two 2FA modes are available: PIN code and Google Authenticator. When 2FA is enabled, login requires email, password, and the second confirmation code.

Telegram usernames must be unique. Changing Telegram resets the previous verification and requires confirmation through the bot again.

Connecting Cryptorg

  1. Create or open a futures Ghost Bot in Cryptorg for the account that will trade through Griders.
  2. Copy the Ghost Bot webhook URL. It usually starts with https://api3.cryptorg.net/crazy/hook/....
  3. Create a Cryptorg API key with read-only access for Bybit Liquidity futures data.
  4. If Cryptorg asks for an IP whitelist, add the Griders server IP.
  5. In Griders, open Connections, add the API key, API secret, webhook URL, and save.
  6. Click Test read-only API. A successful test updates the USDT balance.
  7. Open Strategies, select the same connection, configure GRID DCA 2.9, and save.

For accurate statistics, use a separate Cryptorg account for Griders trades. Manual Cryptorg trades on the same account can make external account history harder to interpret, although Griders monitoring now shows only confirmed Griders deals.

GRID DCA 2.9

GRID DCA 2.9 receives TradingView alerts from the current Pine Script. The script evaluates market stage, ATR, volatility, volume, RSI on 15m and 1h, Bollinger position, BTC/ETH pressure, impulse candles, and global trend diagnostics.

Griders additionally checks plan limits, selected pairs, active positions, repeated-pair protection, personal risk pauses, and system pair pauses before any webhook is sent to Cryptorg.

When a launch is allowed, Griders sends a Ghost Bot payload with first order, DCA grid, take profit, stop loss, fixed x10 leverage, and one trading cycle. Take profit, grid step, and stop are calculated from the signal and GRID DCA presets.

Strategy Settings

Users choose the trading pairs available on their plan. A disabled pair cannot be selected, and Griders will ignore signals for pairs outside the selected list.

Free uses manual first-order input. Plans that support deposit percentage can calculate the first order from the selected share of the connection balance. Manual first order is also capped by the plan and by the current deposit.

Maximum active deals, maximum long deals, and maximum short deals are checked before launch. A value of 0 blocks new launches for that limit.

Signals

The Signals page shows recent events for the selected connection: sent, skipped, and failed. Skipped signals include the reason, such as pair not selected, plan limit reached, active position already exists, risk pause, or trend filter.

TradingView webhook handling is asynchronous: the request is saved immediately, TradingView receives a quick response, and background workers process user launches. Events older than the safety window are treated as stale and are not launched.

Signals linked to confirmed Griders deals are retained so a deal can be traced back to the signal that created it.

Monitoring

Monitoring shows the selected connection balance and active positions through the read-only API, but closed-trade statistics are based only on confirmed Griders deals.

The chart shows cumulative PnL and daily activity for Griders deals only. The default period starts on June 7, 2026, or one day before the first confirmed Griders deal for users who started later.

Dates and times use the time zone selected in Profile.

Risk And Pair Pauses

If a user receives a stop-loss-like close, Griders can pause new launches for that user. Personal risk pauses are shown as danger alerts on Overview.

If a pair has a grid break for another user, Griders can pause new launches for that pair for all users. These system pair pauses are shown as warning alerts and help prevent repeated entries into the same weak setup.

Some pauses can be manually overridden from Overview, but overriding a pause increases risk.

Third-Party Services

Griders depends on Cryptorg, Bybit liquidity, TradingView, Telegram, Tribute, hosting, email providers, and market data APIs. These services have their own rules, fees, outages, limitations, and support procedures.

Paid subscriptions are handled through Tribute. Trading execution and Ghost Bot behavior remain on the Cryptorg side.

Как работает Griders

Что делает Griders

Griders связывает сигналы TradingView с торговлей через фьючерсный Cryptorg Ghost Bot. Сайт принимает сигналы стратегии, проверяет выбранное подключение, применяет лимиты тарифа и защитные фильтры, а затем отправляет команды в Cryptorg через webhook гоуст-бота.

Через read-only API Cryptorg Bybit Liquidity Griders читает баланс, открытые позиции, ордера и закрытый PnL. Торговые команды отправляются только через webhook, который указан в подключении.

Тарифы

Все пользовательские тарифы работают со стратегией GRID DCA 2.9. Старшие тарифы расширяют лимиты одновременных сделок, максимальный первый ордер, доступные пары и способы расчёта объёма.

Бесплатный: действует 14 дней, рекомендуемый депозит от 50 USDT, лимиты 4 / 4 / 4, первый ордер до 6 USDT, ручной ввод первого ордера. Недоступны BTCUSDT, ETHUSDT и SOLUSDT.

Бесплатный Плюс: без ограничения по времени, рекомендуемый депозит от 100 USDT, лимиты 6 / 6 / 6, первый ордер до 12 USDT, расчёт первого ордера от 1% до 10% депозита. Для подключения нужен новый аккаунт Cryptorg, зарегистрированный по реферальной ссылке и подключённый к Griders.

Старт: рекомендуемый депозит от 500 USDT, лимиты 8 / 8 / 8, первый ордер до 60 USDT. Недоступна BTCUSDT. Оплата через Tribute: 10 USDT/мес.

Старт Плюс: рекомендуемый депозит от 1000 USDT, лимиты 10 / 10 / 10, первый ордер до 120 USDT, доступны все пары включая BTCUSDT. Нужен активный Старт и новый реферальный аккаунт Cryptorg.

Премиум: лимиты 12 / 12 / 12, первый ордер до 600 USDT, доступны все пары GRID DCA. Оплата через Tribute: 50 USDT/мес.

Премиум Плюс: лимиты 40 / 20 / 20, первый ордер до 2000 USDT, расчёт первого ордера от депозита до 15%, доступны все пары. Нужен Премиум и новый реферальный аккаунт Cryptorg.

Как подключить платный тариф

  1. Откройте Профиль и укажите Telegram аккаунт без знака @.
  2. Подтвердите Telegram через @griders_tarif_bot.
  3. Откройте Тарифы и подключите нужный платный тариф.
  4. Для тарифов Старт и Премиум оформите подписку через Tribute.
  5. Для Plus-тарифов зарегистрируйтесь по реферальной ссылке Cryptorg и подключите новый реферальный аккаунт к торговле на Griders.

Привязка Telegram нужна для подключения платных тарифов. Если платный тариф не удаётся подтвердить, доступ может вернуться на Бесплатный до восстановления проверки.

Безопасность аккаунта

В Профиле можно указать ник, привязать Telegram, изменить почту, выбрать часовой пояс, отправить запрос на смену пароля и включить двухфакторную аутентификацию.

Доступны два варианта 2FA: PIN-код и Google Authenticator. Если 2FA включена, при входе сайт попросит email, пароль и второй код подтверждения.

Telegram аккаунт должен быть уникальным. При смене Telegram старая привязка сбрасывается, а новый аккаунт нужно подтвердить через бота.

Как подключиться к Cryptorg

  1. В Cryptorg создайте или откройте фьючерсного Ghost Bot на аккаунте, который будет торговать через Griders.
  2. Скопируйте webhook гоуст-бота. Обычно он начинается с https://api3.cryptorg.net/crazy/hook/....
  3. Создайте API ключ Cryptorg с правами только на чтение для Bybit Liquidity futures.
  4. Если Cryptorg требует whitelist, добавьте IP сервера Griders.
  5. В Griders откройте Подключения, добавьте API ключ, API секрет, webhook и сохраните.
  6. Нажмите Проверить read-only API. При успешной проверке Griders обновит баланс в USDT.
  7. Откройте Стратегии, выберите это же подключение, настройте GRID DCA 2.9 и сохраните.

Для объективной статистики лучше выделить отдельный аккаунт Cryptorg только под сделки Griders. Ручные сделки в Cryptorg на том же аккаунте могут мешать разбору общей истории, хотя мониторинг Griders сейчас показывает только подтверждённые сделки Griders.

GRID DCA 2.9

GRID DCA 2.9 получает сигналы TradingView из актуального Pine Script. Скрипт оценивает стадию рынка, ATR, волатильность, объём, RSI на 15м и 1ч, положение в Bollinger Bands, давление BTC/ETH, импульсные свечи и диагностику глобального тренда.

Перед отправкой webhook Griders дополнительно проверяет лимиты тарифа, выбранные пары, открытые позиции, защиту от повторного входа по той же паре, личные паузы риска и системные паузы по торговым парам.

Если вход разрешён, Griders отправляет в Ghost Bot первый ордер, DCA-сетку, тейк-профит, стоп-лосс, фиксированное плечо x10 и один торговый цикл. Тейк, шаг сетки и стоп рассчитываются из сигнала и пресетов GRID DCA.

Настройки стратегии

Пользователь выбирает торговые пары, доступные по тарифу. Недоступную пару выбрать нельзя, а сигнал по паре вне выбранного списка будет проигнорирован.

На бесплатном тарифе используется ручной первый ордер. На тарифах с расчётом от депозита первый ордер может рассчитываться от выбранной доли баланса подключения. Ручной первый ордер также ограничен тарифом и текущим депозитом.

Максимум активных сделок, максимум лонг-сделок и максимум шорт-сделок проверяются перед запуском. Значение 0 означает запрет новых входов по этому лимиту.

Сигналы

В разделе Сигналы отображаются события по выбранному подключению: отправленные, пропущенные и ошибочные. В пропущенных сигналах видна причина: пара не выбрана, достигнут лимит, уже есть открытая позиция, активна пауза риска или сработал трендовый фильтр.

Webhook TradingView обрабатывается асинхронно: событие сразу сохраняется, TradingView быстро получает ответ, а фоновые воркеры запускают проверки и отправку пользователям. Слишком старые события считаются устаревшими и не запускаются.

Сигналы, связанные с подтверждёнными сделками Griders, сохраняются, чтобы сделку можно было связать с исходным сигналом.

Мониторинг

Мониторинг показывает баланс и открытые позиции выбранного подключения через read-only API, но статистика закрытых сделок строится только по подтверждённым сделкам Griders.

График показывает накопленный PnL и дневную активность только по сделкам Griders. Период по умолчанию начинается с 7 июня 2026 года или за день до первой подтверждённой сделки Griders, если пользователь начал позже.

Время сделок и сигналов отображается по часовому поясу, выбранному в профиле.

Паузы риска и паузы пар

Если у пользователя было закрытие, похожее на стоп-лосс, Griders может поставить личную паузу новых запусков. Личные паузы показываются в Обзоре красными уведомлениями.

Если по паре был пробой сетки у другого пользователя, Griders может поставить системную паузу по этой паре для всех. Такие паузы показываются жёлтыми уведомлениями и помогают не повторять входы в слабую рыночную ситуацию.

Часть пауз можно снять вручную из Обзора, но ручной запуск во время паузы повышает риск.

Сторонние сервисы

Griders зависит от Cryptorg, ликвидности Bybit, TradingView, Telegram, Tribute, хостинга, почтовых провайдеров и API рыночных данных. У этих сервисов есть собственные правила, комиссии, ограничения, сбои и поддержка.

Платные подписки оформляются через Tribute. Исполнение сделок и поведение Ghost Bot остаются на стороне Cryptorg.
