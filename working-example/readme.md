# Крипторг Академия

Минимальный PHP-сайт с регистрацией по email, подтверждением аккаунта и защищённой страницей после входа.

## 1) Настройка базы (phpMyAdmin)
> Если при `CREATE DATABASE` возникает ошибка `#1044` (нет прав), это нормально для shared-хостинга: у пользователя часто нет права создавать новые БД.

1. Откройте phpMyAdmin.
2. Выберите **уже существующую** БД, которую выдали хостингом (например, `ctgacademy_r` или похожее имя).
3. Импортируйте файл `db/schema.sql` — он создаёт только таблицу `users` в текущей выбранной БД.
4. Проверьте, что таблицы `users`, `lessons` и `lesson_progress` появились в этой БД.

## 2) Настройка подключения
Отредактируйте `config.php`:
- `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASS`
- `APP_URL` (домен/адрес вашего сайта)

`DB_NAME` должен совпадать с именем БД, к которой у вас реально есть доступ (ту, что выбрали в phpMyAdmin).

## 3) Запуск локально
```bash
php -S localhost:8000
```

## 3.1) Если на сервере ошибка `403 Forbidden (nginx)`
Обычно это значит, что nginx не настроен на `index.php` (или PHP не проксируется в php-fpm).

Минимально проверьте:
- в корне сайта есть `index.php` (главная страница приложения);
- в nginx-конфиге в `index` есть `index.php`;
- настроен блок обработки PHP (`location ~ \\.php$` + `fastcgi_pass`).

## 3.2) Если браузер скачивает `index.php` вместо открытия
Это означает, что PHP на домене не исполняется.

Проверьте в панели хостинга:
- включена версия PHP (например, 8.1/8.2);
- домен привязан к PHP-обработчику (PHP-FPM/CGI);
- для nginx настроена передача `*.php` в `fastcgi_pass`.

## 3.3) Если при регистрации `HTTP ERROR 500`
Проверьте:
- импортирована таблица `users` из `db/schema.sql`;
- в `config.php` верные `DB_HOST/DB_NAME/DB_USER/DB_PASS`;
- на сервере работает отправка почты (`mail()`), либо используйте SMTP-библиотеку.

Если в логах `SQLSTATE[HY000] [2002] Connection refused` или `[2006] MySQL server has gone away`:
- для shared-хостинга чаще всего `DB_HOST = 'localhost'` (а не внешний IP);
- если хостинг выдал Unix socket, укажите его в `DB_SOCKET` и не используйте порт;
- перепроверьте, что `DB_USER` имеет доступ именно к `DB_NAME`.

## 3.4) Настройка отправки писем для верификации
1. Создайте почтовый ящик домена, например `noreply@ctgacademy.ru`.
2. В `config.php` задайте:
   - `MAIL_FROM_EMAIL` — этот адрес ящика;
   - `MAIL_FROM_NAME` — имя отправителя;
   - `MAIL_ENVELOPE_FROM` — обычно тот же `noreply@...`.
3. В DNS домена добавьте/проверьте записи:
   - SPF (разрешить почтовые серверы хостинга),
   - DKIM (включить в панели хостинга),
   - DMARC (рекомендуется).
4. В панели хостинга включите PHP `mail()` для домена (если есть такой переключатель).

## 4) Логика
- Регистрация: email + пароль.
- Отправка письма с ссылкой подтверждения (`verify.php?token=...`).
- Вход доступен только после подтверждения email.
- После входа открывается `/dashboard.php` с приветствием и встроенным видео RuTube.
- В кабинете доступен блок «Курс для начинающих»: карточки уроков берутся из таблицы `lessons`.
- Описание урока кликабельно и ведёт на `/lesson.php?id=...` с полным материалом.
- Отметка о прохождении ставится на странице конкретного урока и сохраняется в `lesson_progress`.

> Важно: для реальной отправки почты настройте SMTP/MTA на сервере (или используйте SMTP-библиотеку).
