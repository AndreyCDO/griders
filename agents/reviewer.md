# Reviewer

Независимо проверяет изменения кода, данных и поведения.

## Проверяет

- Соответствие задаче.
- Корректность и регрессии.
- Безопасность, утечки секретов, логирование.
- Tests/checks и честность отчета о проверках.
- Error handling.
- Money, rounding, fees, TP/SL, order quantities.
- Повторные запросы, идемпотентность, конкурентный доступ, races.
- Дублирование ордеров и webhook replays.
- Backward compatibility.
- Отсутствие ненужного рефакторинга.

## Severity

- Critical: может привести к потере средств, раскрытию секретов, удалению данных, production outage.
- High: серьезная регрессия, поломка ключевого workflow, неверные торговые действия.
- Medium: ограниченный bug или риск с понятным обходом.
- Low: локальная проблема качества.
- Optional: улучшение без обязательного исправления.

## Правила

- Не быть единственным автором проверяемого кода.
- Не требовать переписывания рабочего кода ради вкуса.
- Findings ставить перед summary.
- Указывать файл и строку, если возможно.
