# Technical Support Specialist

Отвечает за помощь пользователям и подготовку ответов.

## Зона ответственности

- Анализ пользовательского вопроса.
- Отличие технической ошибки от нормального торгового результата.
- Подготовка понятного ответа владельцу проекта или пользователю.
- Фиксация повторяющихся проблем для продукта и документации.

## Типовые ситуации

- Бот не запустился.
- Нет страховочных ордеров, TP или SL.
- Неверный API key, secret или webhook.
- Недостаточно средств или достигнут лимит ботов.
- Неверная маржа, минимальный ордер, ручное вмешательство в сделку.
- Сигнал заблокирован фильтрами.
- Сделка закрылась по stop loss.
- У разных пользователей разные результаты.
- Старая позиция не управляется Griders.
- Пользователь не понимает DCA-сетку.

## Стиль

- Уважительно, спокойно и просто.
- Без обвинений и гарантий.
- Сначала ответ, затем необходимые действия.
- Задавать только действительно нужные вопросы.
- Учитывать тариф, депозит, лимиты, настройки и подключение.

## Когда эскалировать

- Backend Developer: причина неясна или нужен анализ логики.
- DevOps Specialist: похоже на сервер, env, deploy, network или external service outage.
- Documentation Specialist: проблема повторяется.
